# 末世監控系統 V2.0 - 改善建議與常見問題

## 一、改善總結

### 1. 高科技末世風格雷達圖

**實現內容**

本系統採用賽博朋克設計美學，打造沉浸式數據監控體驗。雷達圖採用多層同心圓設計，配合旋轉掃描線動畫，呈現科技感十足的末世風格。核心特性包括：

- **多層同心圓**：透明青色邊框構成四層同心圓，代表不同的數據量級
- **旋轉掃描線**：8秒線性旋轉的青色漸變掃描線，營造實時監測感
- **動態數據點**：四個類別（過勞、運動、醫療、其他）以不同顏色標記，配合發光效果
- **中央大數字**：總猝死人數以72px Orbitron字體顯示，配合青色文字陰影

**視覺效果**

雷達圖使用SVG實現，支持響應式縮放。色彩系統採用霓虹發光效果，每個數據點都配有對應顏色的光暈。掃描線使用漸變色（青色→紅色），營造從監測到警告的視覺過渡。

**技術優勢**

- 使用SVG矢量圖形，確保任何屏幕尺寸下都清晰
- CSS動畫實現掃描效果，性能優於Canvas
- 支持懸停交互，提供詳細數據提示

### 2. 響應式布局設計

**三層響應式策略**

系統根據屏幕寬度自動調整布局：

| 設備類型 | 屏幕寬度 | 布局結構 | 特點 |
|---------|---------|---------|------|
| 手機 | < 640px | 單欄堆疊 | 新聞 → 雷達 → 統計 → 明細 |
| 平板 | 640-1024px | 2欄布局 | 左新聞 \| 右雷達+統計 |
| 桌面 | > 1024px | 3欄布局 | 左新聞 \| 中雷達 \| 右統計 |

**自動調整機制**

- 卡片邊距根據屏幕寬度自動縮放（手機0.75rem → 桌面1.5rem）
- 標題字體使用clamp()函數實現流動式縮放
- 表格字體在手機版自動縮小至0.75rem，保持可讀性
- 按鈕在觸屏設備上自動擴大至44px×44px，符合無障礙設計標準

**觸屏優化**

系統檢測觸屏設備，自動禁用懸停效果，改用活動狀態（按下時縮放0.98倍）。這確保在手機和平板上的交互體驗流暢自然。

### 3. 猝死明細表

**數據組織**

猝死明細表位於雷達圖下方，展示詳細的死亡記錄。表格包含以下信息：

- **類別統計**：過勞、運動、醫療、其他四個分類的數量統計
- **記錄摘要**：日期、類別標籤、新聞標題
- **展開詳情**：點擊可展開年齡、性別、地點、詳細描述和原文鏈接

**交互設計**

每條記錄都是可展開的卡片，支持點擊展開/收起。記錄按日期排序，最多顯示前10條，超出部分提示用戶有更多記錄。

**視覺反饋**

每個類別使用對應顏色的邊框和背景，使用戶能快速識別死亡類型。懸停時卡片邊框顏色加深，提供清晰的交互反饋。

## 二、技術亮點

### 設計系統

系統採用統一的設計令牌系統，所有顏色、字體、間距都通過CSS變量定義，便於後續維護和主題切換。

**色彩系統**

```css
--neon-cyan: #00f3ff;      /* 主色 - 科技感 */
--neon-red: #ff003c;       /* 警告色 - 危險 */
--neon-gold: #f5d300;      /* 次色 - 末世感 */
--deep-black: #030305;     /* 背景 - 深空感 */
--text-light: #eeeeee;     /* 正文 - 易讀性 */
```

**排版系統**

- **Orbitron**：標題字體，900字重，未來感十足
- **Share Tech Mono**：技術字體，終端感強
- **Noto Serif TC**：正文字體，中文支持完善

### 動畫系統

系統定義了四種核心動畫，支持reduce-motion媒體查詢以尊重用戶偏好：

| 動畫名稱 | 持續時間 | 用途 |
|---------|---------|------|
| radar-spin | 8秒 | 雷達掃描線旋轉 |
| neon-pulse | 2秒 | 霓虹發光脈衝 |
| danger-blink | 1秒 | 危險狀態閃爍 |
| glitch | 3秒 | 故障文字效果 |

### 無障礙設計

系統遵循WCAG 2.1標準：

- 所有交互元素都支持鍵盤導航
- 顏色對比度達到AAA級（4.5:1以上）
- 支持高對比度模式，自動增加邊框寬度
- 支持減少動畫模式，禁用所有非必要動畫
- 觸屏設備上所有可點擊區域都達到44px×44px最小尺寸

## 三、常見問題（FAQ）

### Q1：如何更新猝死數據？

**A：** 當前系統使用模擬數據。要集成實時數據，需要：

1. 準備數據API端點，返回JSON格式的死亡記錄
2. 在Home.tsx中使用useEffect和fetch替換模擬數據
3. 實現數據緩存和定時更新機制

示例代碼：
```typescript
useEffect(() => {
  fetch('/api/sudden-deaths')
    .then(res => res.json())
    .then(data => setSuddenDeathRecords(data));
}, []);
```

---

### Q2：如何自定義顏色方案？

**A：** 所有顏色都定義在`client/src/index.css`的CSS變量中。修改`:root`選擇器中的顏色值即可全站更新：

```css
:root {
  --neon-cyan: #00f3ff;    /* 修改此行改變主色 */
  --neon-red: #ff003c;     /* 修改此行改變警告色 */
  /* ... */
}
```

---

### Q3：雷達圖如何支持更多數據維度？

**A：** 當前雷達圖支持4個維度（過勞、運動、醫療、其他）。要增加更多維度：

1. 修改CyberRadar.tsx中的categories數組
2. 添加新的顏色定義
3. 更新SVG多邊形計算邏輯

例如，要添加第5個維度，需要將角度間隔從90度改為72度。

---

### Q4：如何在手機上測試響應式設計？

**A：** 使用以下方法：

1. **Chrome DevTools**：按F12打開開發者工具，點擊設備切換按鈕
2. **實機測試**：使用手機瀏覽器訪問開發服務器URL
3. **在線工具**：使用Responsive Design Checker等在線工具

---

### Q5：跑馬燈滾動速度如何調整？

**A：** 在Home.tsx中的CSS部分修改animation持續時間：

```css
@keyframes scroll {
  0% { transform: translateX(100%); }
  100% { transform: translateX(-100%); }
}
.animate-scroll {
  animation: scroll 30s linear infinite;  /* 改變30s為其他值 */
}
```

---

### Q6：如何添加新的聖經資源？

**A：** 修改Home.tsx中的bibleResources數組，添加新對象：

```typescript
{
  id: 'b6',
  title: '新資源標題',
  description: '資源描述',
  url: 'https://example.com',
  language: 'Cantonese',
  category: 'Information',
}
```

### Q7：系統支持哪些瀏覽器？

**A：** 系統支持所有現代瀏覽器：

- Chrome/Edge 90+
- Firefox 88+
- Safari 14+
- 移動瀏覽器（iOS Safari 14+, Chrome Mobile）

不支持Internet Explorer 11及更早版本。

---

### Q8：如何部署到生產環境？

**A：** 按以下步驟進行：

1. 運行`npm run build`生成優化的生產版本
2. 輸出文件位於`dist/`目錄
3. 將dist目錄上傳到Web服務器或CDN
4. 配置服務器以支持客戶端路由（所有路由返回index.html）

---

### Q9：性能優化建議？

**A：** 當前系統已實現以下優化：

- SVG矢量圖形，無需光柵化
- CSS動畫而非JavaScript動畫，性能更優
- 響應式圖片和字體加載
- 掃描線使用CSS而非圖片，減少文件大小

進一步優化可考慮：
- 實現虛擬滾動以處理大量新聞記錄
- 使用Web Workers進行數據處理
- 實現Service Worker進行離線支持

---

### Q10：如何自定義字體？

**A：** 修改`client/index.html`中的Google Fonts導入：

```html
<link href="https://fonts.googleapis.com/css2?family=YOUR_FONT:wght@400;700;900&display=swap" rel="stylesheet" />
```

然後在`client/src/index.css`中更新字體變量：

```css
--font-header: 'YOUR_FONT', sans-serif;
```
